Every entity that appears in a Foundry VTT scene ‚Äî a character, a monster, a trap, a ship ‚Äî is represented by a subclass of `ActorPF2e`. The base class provides the shared foundation: inventory, conditions, rule elements, auras, saving throws, and the damage application pipeline. Concrete actor types extend this foundation with the data and logic specific to their role in the game.

## The `ActorType` union

The system defines all valid actor type strings in `values.ts` and re-exports the union through `types.ts`:

```typescript
// src/module/actor/values.ts
const ACTOR_TYPES = [
  "army",
  "character",
  "familiar",
  "hazard",
  "loot",
  "npc",
  "party",
  "vehicle",
] as const;

// src/module/actor/types.ts
type ActorType = (typeof ACTOR_TYPES)[number];
// => "army" | "character" | "familiar" | "hazard" | "loot" | "npc" | "party" | "vehicle"
```

Use this union whenever you need to constrain or switch over actor types in your own code.

## Checking an actor's type

The simplest way to check a type is a direct string comparison:

```typescript
if (actor.type === "character") {
  // actor is ActorPF2e, not yet narrowed
}
```

<Tip>
  Prefer the `actor.isOfType()` typeguard method ‚Äî it narrows the TypeScript type so subsequent code can access type-specific properties without casting:

  ```typescript
  if (actor.isOfType("character")) {
    // actor is now CharacterPF2e
    console.log(actor.system.details.level.value);
  }

  // Accepts multiple types
  if (actor.isOfType("character", "npc")) {
    // actor is CharacterPF2e | NPCPF2e (both extend CreaturePF2e)
    console.log(actor.system.attributes.hp.value);
  }
  ```

  The method is defined on `ActorPF2e` and backed by the `ActorInstances` interface, which maps each type string to its concrete class.
</Tip>

## The creature subtype

`CharacterPF2e` and `NPCPF2e` both extend the abstract `CreaturePF2e` class rather than `ActorPF2e` directly. `CreaturePF2e` supplies the shared creature logic: senses, speeds, attribute modifiers, strike actions, spellcasting, and skill statistics. `FamiliarPF2e` also extends `CreaturePF2e`.

```typescript
const CREATURE_ACTOR_TYPES = ["character", "npc", "familiar"] as const;
```

You can test for the creature subtype the same way:

```typescript
if (actor.isOfType("creature")) {
  // actor is CreaturePF2e ‚Äî access skills, senses, strikes, etc.
  const perception = actor.perception;
}
```

## Actor classes

<CardGroup cols={2}>
  <Card title="CharacterPF2e" icon="user">
    **Type string:** `"character"`

    Player characters. Extends `CreaturePF2e`.

    Key properties and features:
    - **Ability scores** ‚Äî STR, DEX, CON, INT, WIS, CHA stored in `system.abilities`
    - **Skills** ‚Äî all 16 core skills as `Statistic` instances
    - **Feats** ‚Äî a `CharacterFeats` collection with feat slots for ancestry, class, general, and skill feats
    - **Spellcasting** ‚Äî multiple `SpellcastingEntryPF2e` items forming the actor's `spellcasting` collection
    - **Inventory** ‚Äî `ActorInventory` for all carried physical items
    - **Hero points** ‚Äî tracked in `system.resources.heroPoints`
    - **Crafting** ‚Äî formula book and crafting checks
    - **Automatic Bonus Progression** ‚Äî ABP support that replaces rune economy when enabled
    - **Elemental Blast** ‚Äî kineticist blast calculations via `ElementalBlast`
    - **Level** ‚Äî `system.details.level.value`

    ```typescript
    import { CharacterPF2e } from "@actor";

    declare const actor: CharacterPF2e;
    const level = actor.system.details.level.value;
    const heroPoints = actor.system.resources.heroPoints.value;
    const perception = actor.perception?.mod;
    ```
  </Card>

  <Card title="NPCPF2e" icon="dragon">
    **Type string:** `"npc"`

    Monsters and non-player characters. Extends `CreaturePF2e`.

    Key properties and features:
    - **Flat stat-block layout** ‚Äî saves, AC, perception, and HP are authored directly rather than built from ability scores
    - **Level** ‚Äî `system.details.level.value`; supports Elite (+2) and Weak (‚àí2) adjustments via `isElite` / `isWeak`
    - **Melee items** ‚Äî `MeleePF2e` items represent the NPC's strike actions
    - **Recall Knowledge** ‚Äî DCs calculated via `creatureIdentificationDCs()`
    - **Troop support** ‚Äî `otherSegments` references sibling actors forming a troop
    - **Simple sheet** ‚Äî a streamlined `SimpleNPCSheet` is available as an alternative sheet

    ```typescript
    import { NPCPF2e } from "@actor";

    declare const npc: NPCPF2e;
    const isElite = npc.isElite;
    const baseLevel = npc.baseLevel;
    const description = npc.description; // public notes
    ```
  </Card>

  <Card title="FamiliarPF2e" icon="paw">
    **Type string:** `"familiar"`

    Magical animal companions bonded to a PC master. Extends `CreaturePF2e`.

    Key properties and features:
    - **Master derivation** ‚Äî many statistics (perception, saves, AC) are derived from the bonded master's level and spellcasting proficiency
    - **Master link** ‚Äî `system.master.id` points to the owning character's actor ID
    - **Familiar abilities** ‚Äî feat-like ability items that grant senses, movement types, and special actions
    - **Size** ‚Äî always Tiny unless a familiar ability changes it

    ```typescript
    if (actor.isOfType("familiar")) {
      const masterId = actor.system.master.id;
      const master = game.actors.get(masterId ?? "");
    }
    ```
  </Card>

  <Card title="HazardPF2e" icon="triangle-exclamation">
    **Type string:** `"hazard"`

    Traps and environmental hazards. Extends `ActorPF2e` directly (not a creature).

    Key properties and features:
    - **Stealth DC** ‚Äî `system.attributes.stealth.value` for passive detection
    - **Disable checks** ‚Äî thievery or other skill DCs to disarm
    - **Reset** ‚Äî time and condition to become active again
    - **Routine** ‚Äî action sequence the hazard performs each round when active
    - **Hardness and HP** ‚Äî for complex hazards with destructible components
    - **Reaction** ‚Äî trigger and effect for simple hazards

    ```typescript
    if (actor.isOfType("hazard")) {
      const stealthDC = actor.system.attributes.stealth.value;
      const isComplex = actor.system.details.isComplex;
    }
    ```
  </Card>

  <Card title="VehiclePF2e" icon="ship">
    **Type string:** `"vehicle"`

    Ships, wagons, siege engines, and other conveyances. Extends `ActorPF2e` directly.

    Key properties and features:
    - **Piloting** ‚Äî pilot skill check and DC
    - **Speed** ‚Äî movement speed in feet
    - **Hit points** ‚Äî HP tracked like a creature; no death or dying conditions
    - **Crew** ‚Äî minimum and maximum crew counts
    - **Collision** ‚Äî damage dealt on collision
    - **Size linking** ‚Äî token size can be linked to the vehicle's size attribute

    ```typescript
    if (actor.isOfType("vehicle")) {
      const speed = actor.system.details.speed;
      const pilotingDC = actor.system.details.pilotingCheck;
    }
    ```
  </Card>

  <Card title="PartyPF2e" icon="users">
    **Type string:** `"party"`

    A group entity that tracks party-level resources and member actors. Extends `ActorPF2e` directly.

    Key properties and features:
    - **Member references** ‚Äî `system.details.members` holds the UUIDs of member actors
    - **Travel speed** ‚Äî derived from the slowest member
    - **Camping** ‚Äî rest and daily prep mechanics
    - **Exploration activities** ‚Äî each member's chosen activity during overland travel
    - **No synthetics** ‚Äî `allowSynthetics` returns `false`; parties cannot be unlinked token actors

    ```typescript
    if (actor.isOfType("party")) {
      const memberUUIDs = actor.system.details.members.map((m) => m.uuid);
    }
    ```
  </Card>

  <Card title="ArmyPF2e" icon="chess-rook">
    **Type string:** `"army"`

    Mass combat entities used in the Warfare subsystem. Extends `ActorPF2e` directly.

    Key properties and features:
    - **Morale** ‚Äî current and maximum morale points
    - **Scouting** ‚Äî scouting modifier for strategic movement
    - **Routing** ‚Äî threshold at which the army routs and is removed
    - **Tactics** ‚Äî a list of tactical abilities the army has learned
    - **Gear** ‚Äî equipment items representing army-level gear
    - **Level** ‚Äî army level used to calculate statistics
  </Card>

  <Card title="LootPF2e" icon="chest">
    **Type string:** `"loot"`

    Treasure containers for distributing items to players. Extends `ActorPF2e` directly.

    Key properties and features:
    - **Physical items only** ‚Äî holds weapons, armor, equipment, treasure, etc.
    - **Loot sheet** ‚Äî a dedicated `LootSheetPF2e` for displaying and distributing loot
    - **Observer permissions** ‚Äî players with Observer permission can view but not edit
    - **Pickup** ‚Äî players can transfer items from a loot actor to their own character

    ```typescript
    // Check if an actor is a loot container before transferring items
    if (targetActor.isOfType("loot")) {
      await item.transfer(targetActor);
    }
    ```
  </Card>
</CardGroup>

## Base class properties

All actor types inherit the following from `ActorPF2e`:

| Property | Type | Description |
|---|---|---|
| `inventory` | `ActorInventory` | Collection of all owned physical items |
| `conditions` | `ActorConditions` | Active conditions on this actor |
| `rules` | `RuleElement[]` | Prepared rule elements from all owned items |
| `synthetics` | `RuleElementSynthetics` | Synthetic data generated by rule elements |
| `auras` | `Map<string, AuraData>` | Auras this actor is currently emanating |
| `armorClass` | `StatisticDifficultyClass \| null` | AC statistic difficulty class |
| `spellcasting` | `ActorSpellcasting \| null` | Spellcasting entry collection |
| `saves` | `Record<SaveType, Statistic> \| undefined` | Saving throw statistics |
| `perception` | `PerceptionStatistic \| undefined` | Perception statistic |
| `skills` | `Record<string, Statistic> \| undefined` | Skill statistics |
| `initiative` | `ActorInitiative \| null` | Initiative rolling handler |
